# Grove TUI v1.1 Design

## Goal

Three targeted improvements to the Grove TUI: a scrollable streaming report pane, hierarchical tool-call progress in the activity sidebar during orchestrator runs, and a Grove-green color theme. Fixes the search bar clipping bug discovered at v1.0.

---

## 1. Scrollable Report Pane

**Problem:** `Markdown(id="report")` is a direct child of `Vertical(id="main")`, which does not scroll. Long reports overflow and cannot be read in full.

**Solution:** Wrap `Markdown` in a `VerticalScroll(id="report-scroll")` container. Track `_auto_scroll: bool` on `GroveApp`.

**Scroll behavior (auto-scroll, release on manual scroll):**
- Default: `_auto_scroll = True`. After each `report_chunk` event, if `_auto_scroll` is true, set `_programmatic_scroll = True`, call `report_scroll.scroll_end(animate=False)`, then clear `_programmatic_scroll`.
- Listen for `Scroll` events on the `VerticalScroll`. If the event fires while `_programmatic_scroll` is false (i.e. user-initiated), check `scroll_y < scroll_max_y`; if so, set `_auto_scroll = False` — streaming no longer hijacks the viewport.
- On new query submission (`run_query` entry), reset `_auto_scroll = True` and scroll to top.

**Files changed:** `cli/app.py`, `cli/styles.tcss`

---

## 2. Tool-Call Streaming in Orchestrator Runs

**Problem:** During an orchestrator run, the activity sidebar shows only subagent-level events (`news_macro started / completed`). The individual tool calls within each subagent (e.g. `tavily_news_search`, `yfinance_get_market_data`) are invisible.

**Solution:** Two-layer change — server emits tool events; TUI renders them indented under their parent subagent.

### Server (`server/streaming.py`)

`translate_orchestrator_events` already receives all nested events from `astream_events(version="v2")`. For non-`task` tool events, the tool's parent subagent is identified via `metadata.get("lc_agent_name")` (e.g. `"news_macro_subagent"`).

Add handling for `on_tool_start` / `on_tool_end` where `name != "task"`:
- Strip `_subagent` suffix from `lc_agent_name` to get the parent subagent display name.
- Emit `tool_started` with fields `id`, `tool`, `subagent`, `input`.
- Emit `tool_completed` with fields `id`, `tool`, `subagent`, `duration_s`.

These are the same event types already used by `translate_subagent_events`, now also emitted by `translate_orchestrator_events` with an additional `subagent` field that identifies the parent.

### TUI widgets (`cli/widgets.py`)

Add `ActivityLog.start_nested_item(label: str) -> ActivityItem`:
- Same as `start_item` but prefixes the label with `"  "` (two spaces) and passes a CSS class `"nested"` to the `ActivityItem` for dimmer styling.

### TUI app (`cli/app.py`)

Track `_subagent_items_by_name: dict[str, ActivityItem]` alongside the existing `_items_by_id`.

In `_apply_event`:
- `subagent_started`: create item via `activity.start_item(label)`, store in both `_items_by_id[id]` and `_subagent_items_by_name[name]`.
- `tool_started` (with `subagent` field): call `activity.start_nested_item(label)`, store in `_items_by_id[id]`.
- `tool_started` (without `subagent` field, i.e. direct subagent run): unchanged — `activity.start_item(label)`.
- `subagent_completed` / `tool_completed`: unchanged — look up by `id` in `_items_by_id`, call `mark_done`.
- `error`: mark all in-flight items interrupted, same as before.

On new query in `run_query`: reset both `_items_by_id = {}` and `_subagent_items_by_name = {}`.

**Files changed:** `server/streaming.py`, `cli/widgets.py`, `cli/app.py`

## 3. Green Theme (Grove-Warm)

Palette (Option C from visual brainstorm):
- Background: `#111c14`
- Surface: `#1b2e1e`
- Accent / active elements: `#81c784`
- Body text: `#c8e6c9`
- Border: `#388e3c`
- Input background: `#0d150f`
- Nested item text: `#6a9f6a` (dimmer green for indented tool rows)

In `styles.tcss`, apply the palette by setting background/color directly on specific widgets (Screen, Header, Input, etc.) using hard-coded hex values — more reliable than trying to override Textual's internal theme tokens, which vary by version. Also add a `.nested` CSS class rule that the `ActivityItem` instances created via `start_nested_item` receive, to render dimmer text.

**Files changed:** `cli/styles.tcss`

## 4. Search Bar Clipping Fix

**Problem:** `#prompt` has `dock: bottom` in CSS. `Footer` also docks at bottom. The two compete and push the `Input` partially below the screen edge (bottom border clipped).

**Fix:** Remove `dock: bottom` from `#prompt`. With `Screen { layout: vertical }`, the widget order in `compose()` is:
1. `Header` (auto-docked at top by Textual)
2. `Horizontal(#body, height=1fr)` — fills all space between header and footer
3. `Input(#prompt)` — flows naturally below `#body`
4. `Footer` (auto-docked at bottom by Textual)

No other CSS change needed.

**Files changed:** `cli/styles.tcss`

## Event Schema Changes

`tool_started` and `tool_completed` gain an optional `subagent` field when emitted by `translate_orchestrator_events`. The field is absent when emitted by `translate_subagent_events` (direct subagent runs). The TUI uses presence/absence of `subagent` to choose `start_item` vs `start_nested_item`.

```
# Orchestrator run — tool event (new)
{"event": "tool_started",   "data": {"id": "...", "tool": "tavily_news_search", "subagent": "news_macro", "input": {...}}}
{"event": "tool_completed", "data": {"id": "...", "tool": "tavily_news_search", "subagent": "news_macro", "duration_s": 2.3}}

# Direct subagent run — tool event (unchanged)
{"event": "tool_started",   "data": {"id": "...", "tool": "tavily_news_search", "input": {...}}}
{"event": "tool_completed", "data": {"id": "...", "tool": "tavily_news_search", "duration_s": 2.3}}
```
