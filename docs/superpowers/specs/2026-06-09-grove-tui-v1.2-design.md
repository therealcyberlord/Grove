# Grove TUI v1.2 Design

## Goal

Three targeted visual improvements to the Grove TUI: Grove branding in the header, an animated braille spinner for in-progress activity items, and correct positional grouping of tool calls under their parent subagent during parallel orchestrator runs.

---

## 1. Header Branding

Set `TITLE = "⬡ Grove"` and `SUB_TITLE = "Analyst's CLI"` as class attributes on `GroveApp`. Textual's `Header()` widget renders both automatically — no CSS changes needed.

**Files changed:** `cli/app.py`

---

## 2. Braille Spinner on ActivityItem

Replace the static label display in `ActivityItem` with a timer-driven spinner that animates while the item is in progress.

**Frames:** `⠋⠙⠹⠸⠼⠴⠦⠧⠇⠏` (10 frames, 100ms each → ~1s full cycle)

**Behaviour:**
- `ActivityItem.__init__` stores the bare label (e.g. `"news_macro..."`) in `self._label` and does NOT call `super().__init__` with any content — content is set by the first `_tick`.
- `on_mount` starts an interval timer: `self._timer = self.set_interval(0.1, self._tick)` then calls `self._tick()` immediately so the first frame renders without a 100ms blank.
- `_tick` increments `self._frame` (mod 10) and calls `self.update(f"{SPINNER_FRAMES[self._frame]} {self._label}")`.
- `mark_done(label: str)` stops the timer (`self._timer.stop()`) and calls `self.update(label)`.

**Caller label format:** callers pass the bare display string with no icon prefix — e.g. `"news_macro..."`, `"tavily_news_search..."`. The spinner prepends the animated frame character.

**Files changed:** `cli/widgets.py`

### Indent change

Remove the `"  "` text prefix from `start_nested_item`. Replace with `padding-left: 2;` in the `.nested` TCSS rule. This keeps labels bare for the spinner.

**Files changed:** `cli/widgets.py`, `cli/styles.tcss`

## 3. Parallel Subagent Grouping

During an orchestrator run, multiple subagents execute in parallel. Tool call events from different subagents interleave in the event stream. Tool items must be inserted directly after their parent subagent's last child — not appended to the bottom of the sidebar.

### New tracking state on `GroveApp`

Add `_subagent_last_child: dict[str, ActivityItem]` alongside the existing `_items_by_id`. Reset both on each new query.

### New `ActivityLog` method

```python
def insert_nested_after(self, label: str, after: "ActivityItem") -> "ActivityItem":
    item = ActivityItem(label, classes="nested")
    self.mount(item, after=after)
    return item
```

### Updated `_apply_event` routing

**`subagent_started`:**
- Create item via `activity.start_item(f"{data['name']}...")`
- Store in `_items_by_id[data["id"]]`
- Store in `_subagent_last_child[data["name"]]` (subagent row is its own initial last child)

**`tool_started` with `"subagent"` field (orchestrator run):**
- Look up `last = _subagent_last_child.get(data["subagent"])`
- If `last` exists: call `activity.insert_nested_after(f"{data['tool']}...", after=last)`
- Else: fallback to `activity.start_nested_item(f"{data['tool']}...")`
- Update `_subagent_last_child[data["subagent"]] = item`
- Store in `_items_by_id[data["id"]]`

**`tool_started` without `"subagent"` field (direct subagent run):**
- Unchanged — `activity.start_item(f"{data['tool']}...")`

**`subagent_completed` / `tool_completed`:**
- Unchanged — look up by `data["id"]`, call `mark_done`.

**Files changed:** `cli/app.py`, `cli/widgets.py`

## Label format summary

| Event | Label passed to widget |
|---|---|
| `subagent_started` | `f"{data['name']}..."` |
| `tool_started` | `f"{data['tool']}..."` |
| `subagent_completed` | `f"✓ {data['name']} ({data['duration_s']}s)"` |
| `tool_completed` | `f"✓ {data['tool']} ({data['duration_s']}s)"` |
| `error` | `"✗ interrupted"` |
