# Grove TUI v1.2 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add Grove header branding, an animated braille spinner for in-progress activity items, and correct positional grouping of tool calls under their parent subagent during parallel orchestrator runs.

**Architecture:** Three focused changes to the TUI layer only — no server changes. Header branding via Textual's `TITLE`/`SUB_TITLE` class attributes. Spinner via a per-`ActivityItem` interval timer that animates frames until `mark_done` stops it. Parallel grouping via a `_subagent_last_child` dict that tracks each subagent's last mounted widget so new tool items can be inserted after it using Textual's `mount(after=widget)`.

**Tech Stack:** Python 3.12+, Textual TUI framework (`textual`), `uv` package manager.

- [ ] **Step 1: Rewrite ActivityItem with spinner**

Replace the entire `cli/widgets.py` content:

```python
from textual.containers import VerticalScroll
from textual.timer import Timer
from textual.widgets import Static

_SPINNER_FRAMES = "⠋⠙⠹⠸⠼⠴⠦⠧⠇⠏"


class ActivityItem(Static):
    def __init__(self, label: str, classes: str = "") -> None:
        super().__init__(classes=classes)
        self._label = label
        self._frame = 0
        self._timer: Timer | None = None

    def on_mount(self) -> None:
        self._timer = self.set_interval(0.1, self._tick)
        self._tick()

    def _tick(self) -> None:
        self.update(f"{_SPINNER_FRAMES[self._frame % 10]} {self._label}")
        self._frame += 1

    def mark_done(self, label: str) -> None:
        if self._timer:
            self._timer.stop()
            self._timer = None
        self.update(label)


class ActivityLog(VerticalScroll):
    def clear_items(self) -> None:
        self.remove_children()

    def start_item(self, label: str) -> ActivityItem:
        item = ActivityItem(label)
        self.mount(item)
        item.scroll_visible()
        return item

    def start_nested_item(self, label: str) -> ActivityItem:
        item = ActivityItem(label, classes="nested")
        self.mount(item)
        return item

    def insert_nested_after(self, label: str, after: ActivityItem) -> ActivityItem:
        item = ActivityItem(label, classes="nested")
        self.mount(item, after=after)
        return item
```

Key changes from v1.1:
- `ActivityItem` stores `_label` (bare name, no `◐` prefix) and animates it with the spinner
- `on_mount` starts the interval and fires the first tick immediately
- `mark_done` stops the timer before updating
- `start_nested_item` no longer prepends `"  "` — CSS handles indent
- New `insert_nested_after` uses `mount(after=after)` for positional insertion

- [ ] **Step 4: Update tool_started to use insert_nested_after**

Replace the entire `tool_started` handler in `_apply_event`:

```python
elif event_type == "tool_started":
    data = event["data"]
    label = f"{data['tool']}..."
    if "subagent" in data:
        last = self._subagent_last_child.get(data["subagent"])
        if last:
            item = activity.insert_nested_after(label, after=last)
        else:
            item = activity.start_nested_item(label)
        self._subagent_last_child[data["subagent"]] = item
    else:
        item = activity.start_item(label)
    self._items_by_id[data["id"]] = item
```

How this works: `_subagent_last_child[name]` starts as the subagent row itself. Each tool event for that subagent inserts after the current last child and updates the pointer — so tools stack up directly under their parent, regardless of how other subagents' tools arrive.
